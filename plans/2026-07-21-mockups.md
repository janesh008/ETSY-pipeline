# Implementation Plan: Mockup & PDF Worker (`mockups`)

- **Status:** APPROVED & SAVED
- **Date:** 2026-07-21

## Problem Statement
After upscaling, the clipart files are saved to Google Drive. A mockup generator must compile transparent PNGs from `no_bg/` into preview mockups. A PDF generation script must make the upscaled folder in Google Drive public, retrieve its sharing link, generate a printable PDF catalog referencing this link, and save these artifacts to Google Drive raw data and GCS.

## Proposed Approach
1. Add `share_folder_publicly` and `get_folder_id_by_path` to `GoogleDriveService` in `etsy_pipeline/services/google_drive.py`.
2. Implement `MockupWorker` in `etsy_pipeline/workers/mockup_worker.py`.
3. Define mockup and PDF configuration settings in `etsy_pipeline/workers/mockup_worker_config.py`.
4. Create CLI daemon runner in `scripts/run_mockup_worker.py` to poll MongoDB for jobs pending `mockups`.
5. Wire the worker into `etsy_pipeline/pipeline/orchestrator.py`.
6. Update documentation.
