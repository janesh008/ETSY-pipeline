# Implementation Plan: Background Removal Worker (`bg_removal`)

- **Status:** APPROVED & SAVED
- **Date:** 2026-07-21

## Problem Statement
After image generation, raw PNG clipart images contain backgrounds. Character and prop cliparts (`misc_category`) must have their backgrounds removed using AI (`rembg` with `isnet-general-use`), while pattern, scene, and bonus images (`pattern_scene_bonus_category`) must be copied directly without AI processing. Storage on GCS must be cleaned up post-stage to delete `raw_images/` and conserve storage.

## Proposed Approach
1. Implement `BackgroundRemovalWorker` in `etsy_pipeline/workers/bg_removal_worker.py`.
2. Implement `BackgroundRemovalError` in `etsy_pipeline/utils/exceptions.py`.
3. Define constants in `etsy_pipeline/workers/bg_removal_worker_config.py`.
4. Create CLI daemon runner in `scripts/run_bg_removal_worker.py` to poll MongoDB for jobs pending `bg_removal`.
5. Implement `remove_backgrounds()` function with shared `rembg` session, `clear_gpu()` garbage collection, and direct file copy for pattern/scene categories.
6. Post-stage GCS cleanup: call `gcs.delete_prefix("Clipart/<date>/<theme_slug>/raw_images/")` after successful background removal.
