# Implementation Plan: Upscale Worker (`upscaling`)

- **Status:** APPROVED & SAVED
- **Date:** 2026-07-21

## Problem Statement
Clipart images in `no_bg/` (both characters/props in `misc_category` and patterns/scenes in `pattern_scene_bonus_category`) need to be upscaled to high resolution for printing. The upscaled images must be combined into one single folder and uploaded directly to Google Drive folder `1JWUBqtP-PG-hRLEQj4Kh_vNzfb_G_PCP` (under path `Clipart/main_data/<date>/<theme_slug>/`) rather than GCS. Transparent `no_bg/` images in GCS must NOT be deleted.

## Proposed Approach
1. Add `_get_or_create_folder_by_path` and `upload_folder_to_path` in `GoogleDriveService` (`etsy_pipeline/services/google_drive.py`) to support multi-level nested GDrive folder creation.
2. Implement `UpscaleWorker` in `etsy_pipeline/workers/upscale_worker.py`.
3. Define upscale settings in `etsy_pipeline/workers/upscale_worker_config.py`.
4. Create CLI daemon runner in `scripts/run_upscale_worker.py` to poll MongoDB for jobs pending `upscaling`.
5. Wire the worker into `etsy_pipeline/pipeline/orchestrator.py`.
6. Update documentation.
