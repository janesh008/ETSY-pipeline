## Changes Made

### 1. Unified Theme Slug Handling
- Modified [mockup_worker.py](file:///d:/Janesh/ETSY/ETSY-pipeline/etsy_pipeline/workers/mockup_worker.py) to consistently use `job.theme_slug` (e.g. `Wonder_Woman`, `Lilo_and_Stitch`) instead of direct string replacement `job.theme.replace(" ", "_")` or `job.theme.replace(' ', '_')`.
- This ensures the GCS prefix searches and GDrive uploads successfully match the output directory names generated by upstream stages.

### 2. Subprocess Error Propagation
- Updated the main entry point [main.py](file:///d:/Janesh/ETSY/ETSY-pipeline/etsy%20mockup%20creator/src/main.py) inside the `etsy mockup creator` project.
- The global `except` block now calls `sys.exit(1)` upon failures rather than printing and exiting with code `0`.
- This ensures any errors in mockup creation propagate to `subprocess.run(..., check=True)`, raising exceptions and making failures visible to the orchestrator rather than failing silently with `0` outputs.
