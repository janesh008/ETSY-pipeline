> Phases 8 & 9 of the Etsy pipeline  
> Date: 2026-07-22

---

## Problem Statement

After mockups and the PDF are created, we need to:
1. **Generate SEO-optimized Etsy listing metadata** (title, description, 13 tags) using Gemini 2.5 Flash Vision — fed with mockup PNGs from GCS and the master prompt.
2. **Produce a consolidated per-date CSV file** with all Etsy-required fields, ready for manual review.
3. **Upload live Etsy listings** via the Open API v3 — a separate, human-triggered step.

---

## Two-Shot Architecture

```
 SHOT 1 — Automated daemon (runs after mockups stage)
 ─────────────────────────────────────────────────────────────
 Mockup PNGs on GCS
       │
       ▼
 MetadataWorker  ← Gemini 2.5 Flash Vision
       │  Downloads mockups from GCS
       │  System instruction = Deepseek master prompt
       │  Validates: title ≤140 chars + regex, 13 tags ≤20 chars each
       │  Writes: job.etsy_title, job.etsy_description, job.etsy_tags
       ▼
 CSVWorker
       │  Appends one row to  output/<date>/all_listings.csv
       │  Uploads updated CSV to GCS:  csv/<date>/all_listings.csv
       │  Updates job.csv_path
       ▼
 MongoDB: PENDING etsy_upload → human reviews CSV

 SHOT 2 — Human-triggered (CLI, one theme at a time)
 ─────────────────────────────────────────────────────────────
 python scripts/run_etsy_upload.py --job-id <id>
       │
       ▼
 EtsyWorker
       │  Reads job metadata from MongoDB
       │  POST /v3/.../listings  (create draft)
       │  Uploads mockup images (Hero.png first, then rest alphabetically)
       │  PATCH listing → state=active
       │  Saves listing URL → job.etsy_listing_url
```

## CSV Strategy — Per-Date Consolidation ✅

**Recommended approach: One CSV per date, all themes as rows.**

```
output/
└── 2026-07-22/
    └── all_listings.csv   ← One row per theme processed that day
GCS:
    csv/2026-07-22/all_listings.csv
```

**Why this is better than one CSV per theme:**
- Easy batch review in Excel/Sheets — all listings for the day in one glance
- You can edit multiple listings before any upload
- Single GCS upload path to check each morning
- Etsy's own bulk-upload tool also uses this format (one CSV, many rows)

**Each row = one complete Etsy listing.** The `THEME` and `JOB_ID` columns are internal identifiers for the `run_etsy_upload.py` script to look up the right job.

## Etsy API Constraints (Official, Verified)

### Title
| Rule | Value |
|---|---|
| Max length | **140 characters** |
| Allowed chars | Letters, numbers, punctuation, math symbols, whitespace, `™`, `©`, `®` |
| Regex (block bad chars) | `/[^\p{L}\p{Nd}\p{P}\p{Sm}\p{Zs}™©®]/u` |
| Restricted to once each | `%`, `:`, `&`, `+` |
| Forbidden | `$`, `^`, `` ` ``, emojis, HTML |

### Description
| Rule | Value |
|---|---|
| Max length | **102,400 characters** |
| HTML tags | Not allowed |
| Encoding | UTF-8 |

### Tags
| Rule | Value |
|---|---|
| Count | Exactly **13** |
| Max chars per tag | **20 characters** |
| Format | Multi-word phrases only (no single generic words) |
| No duplicate words | Across all 13 tags |

## Full CSV Schema (All Required Etsy Fields)

```csv
JOB_ID,THEME,TITLE,DESCRIPTION,TAGS,PRICE,QUANTITY,WHO_MADE,WHEN_MADE,
TAXONOMY_ID,TYPE,IS_DIGITAL,MATERIALS,SECTION_ID,
MOCKUP_GCS_PREFIX,PDF_DRIVE_LINK,LISTING_ID,LISTING_URL
```

| Column | Value / Source |
|---|---|
| `JOB_ID` | `job.job_id` — internal, for `run_etsy_upload.py` |
| `THEME` | `job.theme` |
| `TITLE` | `job.etsy_title` — validated ≤140 chars |
| `DESCRIPTION` | `job.etsy_description` — newlines escaped as `\n` |
| `TAGS` | `job.etsy_tags` joined by `\|` (13 tags, each ≤20 chars) |
| `PRICE` | `settings.DEFAULT_LISTING_PRICE` (e.g., `3.99`) |
| `QUANTITY` | `settings.DEFAULT_LISTING_QUANTITY` (e.g., `999`) |
| `WHO_MADE` | `i_did` *(AI generated by me)* |
| `WHEN_MADE` | `made_to_order` |
| `TAXONOMY_ID` | Fetched from Etsy API at worker init — node for "Clip Art" |
| `TYPE` | `download` |
| `IS_DIGITAL` | `true` |
| `MATERIALS` | `PNG,Digital Download,Transparent Background` |
| `SECTION_ID` | `""` (optional — shop section, can leave blank) |
| `MOCKUP_GCS_PREFIX` | GCS path prefix where mockups live |
| `PDF_DRIVE_LINK` | `job.pdf_drive_link` |
| `LISTING_ID` | Filled after Shot 2 |
| `LISTING_URL` | Filled after Shot 2 |

> [!NOTE]
> `SECTION_ID` can be left blank initially and filled manually in the CSV review if you want to assign the listing to a shop section.

```python
TAG_MAX_CHARS = 20

```

#### [NEW] [metadata_worker.py](file:///d:/Janesh/ETSY/ETSY-pipeline/etsy_pipeline/workers/metadata_worker.py)
**Flow:**
1. Download all mockup PNGs from GCS prefix `mockups/<date>/<slug>/`
2. Build Gemini multimodal request:
   - System instruction: full text of `Deepseek_etsy_listing_generator_prompt.txt`
   - User turn: `"Generate the complete Etsy listing for this clipart bundle."`
   - Image parts: all mockup PNGs as `inline_data` bytes
3. Parse Gemini response → extract `ETSY TITLE`, `ETSY DESCRIPTION`, `ETSY TAGS` blocks
4. **Validate & sanitize:**
   - Strip forbidden chars from title using regex
   - Assert `len(title) <= 140`
   - Assert each restricted char appears ≤1 time in title
   - Assert exactly 13 tags, each `len(tag) <= 20`
   - Strip description of any HTML tags
5. Store to `job.etsy_title`, `job.etsy_description`, `job.etsy_tags`
6. Upload raw Gemini response to GCS `metadata/<date>/<slug>/raw_response.txt`

### Phase 8b — CSVWorker

#### [NEW] [csv_worker.py](file:///d:/Janesh/ETSY/ETSY-pipeline/etsy_pipeline/workers/csv_worker.py)
**Key design:**
- CSV is **per-date, shared across all themes**
- File path: `output/<date>/all_listings.csv`
- GCS path: `csv/<date>/all_listings.csv`
- Uses Python `csv` stdlib — no pandas dependency
- **Atomic append**: download existing CSV from GCS → append row → re-upload
  (safe for sequential daemon runs, not concurrent)

**Image ordering for `MOCKUP_GCS_PREFIX`:**  
When EtsyWorker later uploads images, it sorts them with `Hero.png` (case-insensitive) always first:
```python
def sort_mockup_images(paths: list[str]) -> list[str]:
    return sorted(paths, key=lambda p: (0 if Path(p).stem.lower() == "hero" else 1, p))
```

### Phase 9 — EtsyWorker + Scripts

#### [NEW] [scripts/etsy_oauth.py](file:///d:/Janesh/ETSY/ETSY-pipeline/scripts/etsy_oauth.py)
One-time interactive OAuth 2.0 PKCE flow:
```
python scripts/etsy_oauth.py
→ Opens browser: https://www.etsy.com/oauth/connect?...
→ User authorizes → callback → saves ETSY_ACCESS_TOKEN + ETSY_REFRESH_TOKEN to .env
```

#### [NEW] [etsy_worker.py](file:///d:/Janesh/ETSY/ETSY-pipeline/etsy_pipeline/workers/etsy_worker.py)
**Flow:**
1. Refresh OAuth token if expired (using `ETSY_REFRESH_TOKEN`)
2. Fetch `taxonomy_id` for "Clip Art" via `GET /v3/application/seller-taxonomy/nodes`
3. `POST /v3/application/shops/{shop_id}/listings` — create draft
4. Upload mockup images **in order: Hero.png first, then alphabetically**
   - Each image: `POST /v3/application/shops/{shop_id}/listings/{id}/images`
   - Max 10 images per listing (Etsy limit)
5. `PATCH` listing → `state=active`
6. Save `job.etsy_listing_id`, `job.etsy_listing_url`
7. Update the CSV row with `LISTING_ID` + `LISTING_URL` (re-downloads, updates, re-uploads)

#### [NEW] [scripts/run_metadata_worker.py](file:///d:/Janesh/ETSY/ETSY-pipeline/scripts/run_metadata_worker.py)
Daemon: polls MongoDB for `mockups=COMPLETED` + `metadata_generation=PENDING`, runs MetadataWorker + CSVWorker.

#### [NEW] [scripts/run_etsy_upload.py](file:///d:/Janesh/ETSY/ETSY-pipeline/scripts/run_etsy_upload.py)
**Manual trigger only.** Usage:
