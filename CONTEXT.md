# CONTEXT.md — Repository Root

**Last reviewed:** 2026-07-19

## Responsibility
End-to-end Etsy digital-product asset pipeline: takes a cartoon theme as input, generates AI image prompts, produces PNG clipart images, removes backgrounds, upscales, builds product mockups, generates Etsy listing metadata, exports a CSV, and uploads listings — without any notebook-specific code.

## Not responsible for
Running the legacy Colab notebooks in `ETSY_main_colab/` or the code in `etsy mockup creator/`. Both of these are separate git repositories and are completely ignored by this repository's Git setup. This repository is dedicated solely to the `etsy_pipeline` project and its supporting scripts, tests, and configurations.

## Public interface (entry points)
- **CLI:** `python -m scripts.run_prompts --theme "X" --event birthday`
- **Programmatic:** `from etsy_pipeline.pipeline.orchestrator import Pipeline; Pipeline().run(job)`
- **Graph:** `.repo-graph/graph.json` — read this first in any agent session for structural context

## Dependencies (top-level)
- `google-genai` — Vertex AI / Gemini 2.5 Flash calls
- `pydantic` / `pydantic-settings` — data models and config
- `python-dotenv` — `.env` file loading
- `ruff`, `mypy`, `pre-commit` — developer tooling

## Gotchas / invariants
- **Never import from `ETSY_main_colab/`** — it contains Colab-specific code and is not a proper Python package.
- Auth mode is toggled by `USE_VERTEX_AI` in `.env`. Both `GOOGLE_API_KEY` and Vertex AI ADC paths are supported in the same binary.
- `.env` is gitignored; `.env.example` is the canonical template.
- The code graph at `.repo-graph/graph.json` is committed and auto-regenerated by pre-commit. Never edit it by hand.
- All hardcoded values must live in `etsy_pipeline/config/settings.py` — no magic strings in workers.
